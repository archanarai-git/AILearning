# Part 1: Core Foundations

**Learning Time:** 8-12 hours with exercises
**Difficulty:** 🟢 Beginner → 🟡 Intermediate
**Goal:** Master the fundamentals of AI-assisted development for technical interviews

---

## Overview

Part 1 builds your foundation in AI-assisted development. You'll learn:

✅ When and how to use AI coding assistants effectively
✅ Prompt engineering techniques that work across all tools
✅ Systematic iteration and refinement workflows
✅ Critical code review and validation skills
✅ Common pitfalls and how to avoid them

**Prerequisites:**
- Basic programming knowledge in at least one language
- Access to an AI coding assistant (Claude Code, Copilot, Cursor, etc.)
- Willingness to practice and iterate

---

## Chapters

### [Chapter 1: Introduction to AI-Assisted Development](chapters/chapter-01-introduction.md) 🟢
**Time:** 1.5-2 hours
**Focus:** Setting context and foundational mindset

Learn what AI coding assistants can and cannot do, when to use them, and what interviewers are really evaluating when you use AI in interviews.

---

### [Chapter 2: Prompt Engineering Fundamentals](chapters/chapter-02-prompt-engineering.md) 🟢→🟡
**Time:** 2-3 hours
**Focus:** Writing effective prompts

Master the art of crafting clear, specific prompts that get you the code you need. Learn prompt patterns for implementation, debugging, optimization, and more.

---

### [Chapter 3: Effective Iteration Workflows](chapters/chapter-03-iteration-workflows.md) 🟡
**Time:** 2-3 hours
**Focus:** Systematic refinement

Learn the iteration cycle: Generate → Review → Refine → Validate. Understand when to iterate vs. start over, and how to manage time effectively.

---

### [Chapter 4: Code Validation & Review](chapters/chapter-04-code-validation.md) 🟡
**Time:** 2-3 hours
**Focus:** Critical evaluation skills

Develop systematic code review practices. Learn to quickly read, understand, and validate AI-generated code for correctness, efficiency, readability, and security.

---

### [Chapter 5: Common Anti-Patterns & Pitfalls](chapters/chapter-05-anti-patterns.md) 🟡
**Time:** 1.5-2 hours
**Focus:** What NOT to do

Learn from real failures. Understand common mistakes like over-reliance, under-utilization, prompt paralysis, and iteration hell. Discover recovery strategies.

---

## Learning Path

**Recommended Order:**
```
Chapter 1 → Chapter 2 → Chapter 3 → Chapter 4 → Chapter 5
```

Each chapter builds on previous concepts. Exercises reinforce learning.

**Time Commitment:**
- **Fast track:** 8 hours (reading + key exercises)
- **Thorough:** 12 hours (reading + all exercises)
- **Mastery:** 15+ hours (reading + exercises + projects)
